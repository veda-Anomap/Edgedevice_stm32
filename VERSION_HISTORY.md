## [v1.2.1] - 2026-03-20
### 외부/내부 워치독 튜닝 + 큐 복구

변경 전 문제
- IWDG 타임아웃(약 5초) 대비 모니터 miss 임계가 빡빡해 지터 상황에서 불필요 리셋 위험이 있었음.
- `.ioc` 재생성 중 `control_queue`가 `64 -> 16`으로 줄어 burst 내성이 약해짐.

변경 이유
- 외부 릴레이 워치독과 내부 IWDG의 실전 안정성을 높이기 위함.
- UART 명령 버스트 구간에서 드롭/ACK fail 가능성을 낮추기 위함.

구현 방식
- `SYS_MONITOR_MISS_MAX`를 `4`로 조정.
- `StartDefaultTask` 워치독 대기 구간을 helper 함수로 정리.
  - `rpi_wdg_wait_seconds_and_mark_alive()`
  - `rpi_wdg_sync_heartbeat_baseline()`
- `control_queue`를 코드/IOC 모두 `64`로 복구.
- README에 워치독 정책과 큐 길이 반영.

변경 후 효과
- 워치독 오탐 리셋 여유 증가.
- 릴레이 컷/복구/재-grace 흐름의 가독성 향상.
- burst 명령 수용성 복구.

영향 파일
- `Core/Src/main.c`
- `irrled.ioc`
- `README.md`

## [v1.1.4] - 2026-03-19
### 수동 명령 버스트 완화 + 서보 안전 범위 축소

변경 전 문제
- `w/a/s/d` 명령이 몰리면 서보가 급격히 연속 동작.
- 종단 근처 충돌/과동작 가능성.

변경 이유
- 실제 하드웨어에서 급격한 움직임과 끝단 충돌을 줄이기 위함.

구현 방식
- 이동 명령 coalescing(최신 1개 유지).
- `MANUAL_CMD_MIN_INTERVAL_MS`로 적용 간격 제한.
- PAN/TILT PWM 한계를 안쪽으로 축소.

변경 후 효과
- 급격한 왕복 감소.
- 종단 충돌 가능성 감소.

영향 파일

## [v1.1.3] - 2026-03-18
### UART1 파서 강건화(프레임 경계/타임아웃/큐)

변경 전 문제
- 임의 바이트가 CMD로 해석될 가능성.
- 프레임 조립 중 입력 끊김 시 상태 꼬임.
- 단일 완료 버퍼 구조에서 버스트 유실 위험.

구현 방식
- CMD allow-list(`0x04/0x05`) 적용.
- `PROTO_RX_TIMEOUT_MS` 기반 조립 타임아웃 리셋.
- 완료 프레임 큐 다중 슬롯화.
- 바이트 조립과 프레임 처리 경로 분리.

변경 후 효과
- 버스트 내성 향상.
- 파서 복구 속도 개선.

영향 파일

- `Core/Src/main.c`

## [v1.0.0] - 2026-03-18
### 마이크 경로 ADC -> I2S 전환

변경 전 문제
- ADC/I2S 경로 혼재로 기준점/노이즈 처리 불명확.

- 동적 DC 오프셋 추적 + 노이즈 게이트 적용.
- short/long window + SNR/ratio 게이트 유지.

변경 후 효과
- 저레벨 노이즈 오탐 감소.
- 기존 제어 로직과의 호환 유지.

영향 파일

- `Core/Inc/mic.h`

- `irrled.ioc`

## 참고: Gemini/Codex 계획 반영 방향

- ISR 최소화: 인터럽트에서는 큐잉만, 파싱/응답은 태스크에서 처리.
- 태스크 분리: 제어 루프/센서 루프를 분리해 지연 전파 차단.
- 프로토콜 강건성: CMD 검증, LEN 검증, 타임아웃, 버스트 흡수.
- 진단 강화: UART 오류 카운터 분리로 원인 분석 시간 단축.
- 모터 안정성: burst coalescing + 최소 적용 간격 + 안전 범위 적용.
- 신호 안정성: I2S DC 블로킹/노이즈 게이트 + 윈도우 기반 SNR 게이트 유지.
